# DevFlow v0.1.0 MVP Preview

DevFlow v0.1.0 is the first public MVP preview of a local-first AI workflow for frontend delivery.

It helps developers turn requirements, UI notes, API documentation, and existing repository signals into reviewable planning, implementation, verification, visual, and delivery artifacts.

## Highlights

- Local-first `dev-flow` CLI for frontend repositories.
- Structured starter documents for requirements, UI notes, and API docs.
- Project brief generation from source documents and repository stack signals.
- Requirements extraction for user stories, constraints, and acceptance criteria, including checkbox items plus bullets, numbered lists, Given/When/Then lines, simple table rows, and localized acceptance sections.
- UI design asset extraction from Markdown image links and recognized design handoff links, including local SVG metadata, color swatches, text snippets, and PNG/JPEG dimensions.
- UI design token extraction for colors, typography, spacing, radius, shadows, motion, and iconography.
- UI state checklist extraction from UI notes, including screens, components, states, interactions, responsive behavior, and accessibility checks.
- Normalized frontend targets for routes/views, components, data needs, and UI states in project briefs, task plans, and AI execution target profiles, including targets derived from acceptance criteria.
- Delivery risk scoring for ambiguous requirements, missing UI/API detail, missing verification commands, and unresolved project gates.
- API extraction for HTTP endpoints, inline query/path parameters, GraphQL operations and linked schemas, JSON and GraphQL data models, API-driven loading/cache/refresh/data freshness requirements, fenced or locally linked OpenAPI/Swagger JSON/YAML paths, parameters, schemas, auth, security, errors, and response details.
- Expanded stack detection from package manager metadata, workspace packages, nested frontend source directories, React Router conventions, and common framework/tool config files.
- Aggregated recommended verification commands from detected root/workspace `check`, `lint`, `format:check`, `typecheck`, unit/component/integration/E2E test, coverage, `build`, and audit package scripts, common local/CI aliases, or inferred quality, framework type-checking, test, and build tooling.
- Implementation planning with deterministic fallback output and OpenAI-compatible provider support.
- Frontend delivery blueprint sections in implementation plans for routes, components, state, data/API integration, styling, tests, and accessibility.
- Explicit frontend route path and component name extraction from requirements, UI notes, user stories, and acceptance criteria.
- Planner guardrails to keep patch-set JSON out of implementation-plan artifacts and append the generated blueprint when provider plans omit it.
- Task generation with implementation units derived from explicit route/view targets, component targets, frontend data needs, frontend state targets, requirements, UI, UI state checklist items, design assets, design tokens, API contracts, and API-driven state requirements, with dependency hints and type-specific review checklists for focused handoff.
- AI-assisted dry-run patch proposals for review before source-changing execution, including UI checklist coverage and delivery risk summaries.
- Source-context target profile candidates derived from explicit route paths and component names in requirements and UI notes, with selected frontend route/component units prioritized when `--unit` is used.
- Data/API target profile candidates derived from explicit API endpoint paths, with selected frontend data units prioritized when `--unit` is used.
- Selected-unit-prioritized source-context sampling so AI prompts see likely route, component, or data files before broader repository candidates.
- Source-context sampling summaries in `.devflow/artifacts/source-context-summary.json`, delivery reports, and delivery manifests so reviewers can audit which path-level context AI prompts used.
- Explicit prompt audit artifacts for planner, dry-run, patch-set, and full delivery flows through `--save-prompt` and `deliver --save-prompts`.
- Doctor JSON diagnostics for saved prompt artifact counts and latest prompt metadata.
- Doctor JSON diagnostics for AI provider mode, key source, normalized endpoint, model, default/env sources, and fixture override state without exposing provider keys.
- Nuxt-aware execution target profiles for route, layout, composable, server API, style, Playwright, and Cypress candidates.
- Svelte/SvelteKit, Astro, and Angular-aware execution target profiles for route, data/API, service, style, and spec candidates.
- Validated patch-set application with write, replace, guarded delete, operation limits, execution logs, task changelogs with review handoff links, reviewer-authored notes, verification summary refreshes, backups, rollback, and automatic restore after partial apply failures.
- Optional clean-worktree guardrails for source-changing apply through `execute --apply --require-clean`, `deliver --apply --yes --require-clean`, and the GitHub Action `require-clean` input.
- Safe `deliver` orchestration for planning, task generation, dry-run proposals, optional explicit apply, verification, visual checks, and final reports.
- Visual checks for desktop, tablet, and mobile screenshots, blank-screen detection, required text, inferred delivery text checks from design assets, acceptance criteria, and UI state brief context, horizontal overflow, clipped text, and overlapping visible elements.
- Delivery reports with source context, prompt artifact directory status, acceptance criteria, source-signal matched per-criterion evidence, UI state checklist items, design tokens, API state requirements, risk assessment, design/API details, task changelog review handoff notes, changelog verification summaries, verification failure excerpts, remediation hints, structured remediation plans, embedded visual evidence, touched files, operation counts, backup counts, and delivery readiness.
- Delivery manifest JSON with machine-readable artifact status, prompt artifact directory status, readiness, verification, visual, source-change, criterion-specific acceptance evidence, applied operation details, visual layout issue details, task-changelog handoff, design-token, API-state, screenshot, touched-file, backup, verification-failure, remediation, remediation-plan, open-question, and delivery-risk summaries.
- Delivery manifest JSON schema for CI, editor, dashboard, and downstream-agent integrations, including API state requirement evidence, applied operation details, and visual layout issue details.
- Local `dev-flow status` summaries and CI gate exits backed by delivery manifests, including applied operation counts, required visual text missing counts, visual layout issue counts, API state requirement counts, and live provider smoke evidence in text output when the smoke report exists.
- Composite GitHub Action for non-destructive CI delivery by default, tested manifest-backed job summaries and readiness, verification, and visual gates, artifact upload evidence on failed runs, and double confirmation for source-changing delivery.
- React/Vite dashboard example with fixture-backed source-changing delivery.
- Public GitHub install support before the first npm publish.
- Automated npm package contents check to prevent generated artifacts, dependencies, and test outputs from entering the tarball.
- Installed CLI live-provider smoke checks plus static release readiness checks for package metadata, changelog coverage, release notes, npm provenance workflow configuration, validated machine-readable live provider smoke evidence, sanitized GitHub job-summary evidence, and Release workflow artifact retention.

## Install

Before npm publish:

```bash
npm install --global github:WangXueZhi/dev-flow-ai
dev-flow help
```

After npm publish:

```bash
npm install --global dev-flow-ai
dev-flow help
```

## Quick Trial

```bash
dev-flow init
dev-flow brief
dev-flow plan
dev-flow tasks
dev-flow execute --dry-run
dev-flow verify
dev-flow report
dev-flow deliver
```

For a preview URL:

```bash
dev-flow deliver \
  --preview-url http://127.0.0.1:5173 \
  --visual-text Home,Checkout
```

## Release Boundary

This is an MVP preview for local workflow trials, examples, and contributor development.

Known release gates before publishing this draft:

- Configure `NPM_TOKEN` with publish rights for `dev-flow-ai`.
- Configure `DEVFLOW_AI_API_KEY` or `OPENAI_API_KEY` so GitHub Release publication can pass the required live provider smoke.
- Run `npm run smoke:live` with a real provider key.
- Verify the Release workflow uploaded the `live-provider-smoke-report` artifact with `status: "passed"`.
- Publish `dev-flow-ai@0.1.0` to npm.
- Verify `npm install --global dev-flow-ai`.
